# 팩플로우 수동 QA 체크리스트

## 1. 로그인 / 권한

- 관리자 계정으로 로그인된다.
- 사무/영업 계정은 일반 데이터 저장이 된다.
- 현장 작업자 계정은 일반 `/api/data` 쓰기가 막힌다.
- 관리자 외 계정은 백업/복원 화면 또는 API 접근이 막힌다.
- 관리자 외 계정은 `ino_users` 같은 민감 데이터 접근이 막힌다.

## 2. 견적 → 수주

- 패키지 견적 저장이 된다.
- `→ 패키지 수주` 전환 후 수주 목록에 새 수주가 보인다.
- 견적에 `orderId`가 연결된다.
- 수주 금액과 품목 정보가 견적과 일치한다.

## 3. 수주 → 작업지시

- 단일 품목 수주에서 작업지시 생성이 된다.
- 다품목 수주에서 품목 선택 후 각각 별도 WO 생성이 된다.
- 수주 상세에 `woIds`, `woLinks`가 누적된다.
- WO 저장 후 수주 상태가 `생산중`으로 바뀐다.

## 4. 작업지시

- 거래처/품목/협력사가 자동 등록된다.
- 공정 저장 후 외주 공정이 외주관리와 동기화된다.
- 작업지시 수정 후 외주 수량/납기 변경이 반영된다.

## 5. 입고 → 재고 / 매입

- 입고 저장 후 재고가 생성 또는 증가한다.
- 입고 저장 후 매입 레코드가 자동 생성된다.
- 입고 수정 후 재고/매입 수량과 단가가 같이 갱신된다.
- 입고 삭제 후 재고/매입 반영이 같이 제거된다.

## 6. 출고 → 매출 / 세금계산서

- 출고 저장 후 `shipLog`가 생성된다.
- 출고 저장 후 `sales`가 자동 생성된다.
- 출고 저장 후 `taxInvoice`, `etax`가 자동 생성된다.
- 거래처 사업자번호가 있으면 세금계산서 method가 `전자대기`가 된다.
- 불량 수량이 있으면 `qcRecords`가 생성된다.
- 부분 출고 시 WO 상태가 `출고대기`로 유지된다.
- 전량 출고 시 WO와 수주 상태가 `출고완료`가 된다.
- 출고 상세에서 `출고 취소`를 누르면 `shipLog`, `sales`, `taxInvoice`, `etax`, `qcRecords`가 함께 정리된다.
- 출고 취소 후 WO와 수주 상태가 출고 전 단계로 되돌아간다.

## 7. 백업 / 복원

- 수동 백업 생성이 된다.
- 백업 목록이 보인다.
- 다운로드 백업 파일이 JSON으로 열리고 `_backup` 마커가 있다.
- 업로드 복원이 된다.
- 저장된 백업 파일 복원이 된다.

## 8. UI / 동선

- 수주 목록에서 진행 흐름, 잔량, 연결 WO 정보가 보인다.
- 출고 상세에서 매출/세금계산서/전자세금 연동 현황이 보인다.
- 패키지 견적 모달에서 최근 견적/최근 단가/현재 계산 기준이 보인다.
- 수주 등록 화면에서 품목 수, 총 수량, 예상 금액, 최근 수주 힌트가 보인다.
- 작업지시 화면에서 연결 수주번호, 견적번호, 수량/납기 요약이 보인다.
- 출고 준비 목록에서 자체생산/협력사, 수주 연결, 문서 대기 여부가 보인다.

## 9. 패키지 제조 실무 체크

- 같은 거래처/같은 품목 반복 주문 시 최근 사양과 최근 단가를 참고해 입력할 수 있다.
- 작업지시 복사 후 납품예정일과 수량만 바꿔서 재사용할 수 있다.
- 외주 작업이 있는 WO는 협력사 정보가 눈에 띄게 보인다.
- 납기 임박 또는 지연 WO가 수주/출고 화면에서 구분된다.
- 부분 출고 후 잔량이 다음 출고 준비 목록에 이어진다.

## 10. 운영 점검

- 저장 충돌 발생 시 서버값 유지/내 변경 다시 저장 모달이 뜬다.
- 충돌 모달에서 재시도 후 정상 저장된다.
- 관리자 화면에서 숨긴 비패키지 메뉴가 다시 노출되지 않는다.
