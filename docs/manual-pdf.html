1. 시작하기

1-1. 로그인

로그인 방법
  1. 브라우저에서 http://localhost:8080 접속
  2. 사용자 선택 → 본인 계정 선택
  3. 비밀번호 입력 (초기: 1234)
  4. 로그인 클릭

1-2. 화면 구성

영역설명
왼쪽 사이드바6개 카테고리 메뉴 (영업관리, 생산관리, 품질/설비, 자재/구매, 회계/인사, 시스템). 카테고리 클릭 → 하위 메뉴 펼침 → 클릭 시 오른쪽에 해당 화면 표시
상단 헤더현재 위치 표시 (카테고리 > 메뉴명), 퀵 버튼들, 그룹탭
오른쪽 콘텐츠선택한 메뉴의 실제 내용 — KPI 카드, 데이터 테이블, 등록/수정 기능

상단 퀵 버튼

버튼기능
+ 작업지시서어느 화면에서든 바로 작업지시서 생성
알림 (종)안전재고 부족, 납기 임박 등 시스템 알림
거래처 / 품목 / 목형빠른 등록·검색 팝업
KPI실시간 경영 지표 화면 이동
납기납기 3일 이내 수주 긴급 확인

그룹탭

메뉴 이름 옆에 표시되는 탭 버튼입니다. 관련 기능 간 빠른 전환이 가능합니다.

예) 설비관리 진입 시: 설비대장 예방보전 고장이력 비가동 OEE 금형이력

1-3. 처음 사용할 때 — 기초 데이터 등록 순서

기초 데이터가 없으면 작업지시서, 견적, 출고 등에서 거래처/품목을 선택할 수 없습니다. 반드시 아래 순서대로 먼저 등록하세요.
1. 회사정보→ 2. 직원 등록→ 3. 거래처 등록→ 4. 품목 등록→ 5. 목형 등록→ 6. 설비 등록
순서위치입력 내용
1. 회사정보환경설정 > 설정회사명, 대표자명, 사업자등록번호, 업태, 종목 (세금계산서에 사용됨)
2. 직원회계/인사 > 인사관리이름, 부서(현장/사무), 직급, 담당공정(코팅/합지/톰슨/접착/인쇄), 기본급
3. 거래처기초정보 > 거래처거래처명, 사업자등록번호, 연락처, 주소, 유형(납품처/매입처)
4. 품목기초정보 > 품목품목명, 규격, 재질, 단가 (예: "피자박스 / 400x300x50 / E골 / 350원")
5. 목형기초정보 > 목형목형명, 거래처, 사이즈, 상태 (톰슨 공정에 사용하는 금형)
6. 설비품질/설비 > 설비관리 > 설비대장설비명(코팅기 1호 등), 제조사, 구매일, 상태

2. 영업관리

고객 문의부터 견적 → 수주 → 출고까지의 전체 영업 흐름을 관리합니다.

견적 작성→ 수주 확정→ 작업지시서 생성→ 생산 완료→ 출고 처리

2-1. 견적관리

거래처에서 제품 제작 문의가 오면 견적을 작성합니다.

견적 등록 순서
  1. 영업관리 > 견적관리 이동
  2. + 견적등록 클릭 → 견적번호 자동 부여 (QT-20260411-001)
  3. 거래처: 요청한 거래처 (자동완성 지원)
  4. 제품명: 만들 제품 이름 (예: "피자박스 대")
  5. 수량: 주문 수량 (매 단위)
  6. 확정 공급가: 최종 판매 단가 (마진 포함)
  7. 클로드 견적 내용: AI 원가 계산 결과 붙여넣기 (선택사항)
  8. 저장

견적 상태 흐름

상태의미다음 행동
작성중아직 거래처에 미전달내용 확인 후 거래처에 전달
발송거래처에 견적서 전달 완료거래처 회신 대기
수주거래처가 주문 확정→ 작업지시서 전환 버튼 클릭
실주수주 실패원인 기록 후 종료

"클로드 견적 내용" 칸이란?

AI(클로드)에게 "E골 400x300x50 박스 1000매 원가 계산해줘"라고 물어본 결과를 통째로 복사해서 붙여넣는 곳입니다. 원지비, 코팅비, 합지비, 톰슨비, 접착비 등의 원가 breakdown이 기록으로 남습니다.

연동·동기화
  • 견적 → 수주: 상태를 "수주"로 변경하면 수주관리에 자동 등록
  • 견적 → 작업지시서: "작업지시서 전환" 버튼으로 견적 정보가 작업지시서에 자동 복사
  • 견적 → 인쇄: "인쇄" 버튼으로 공식 견적서 양식 출력

2-2. 수주관리

확정된 주문을 관리합니다.

그룹탭기능
수주목록전체 수주 현황. 상태별 필터링 (진행중/완료/취소)
진행추적수주별 현재 단계 추적 — 생산 진행률, 출고 여부 확인
연동·동기화
  • 수주 → 작업지시서: 수주에서 바로 작업지시서 생성 가능
  • 수주 → 납기관리: 납기일 기반 알림 자동 생성 (3일 전 경고)
  • 수주 → 출고: 해당 작업지시서 완료 시 출고 가능 목록에 연결

2-3. 출고관리

생산 완료된 제품을 거래처에 납품합니다.

그룹탭기능
출고작업지시서 "완료" 건만 표시. 출고 수량/차량번호 기록
부분출고전체 수량 중 일부만 먼저 출고 (예: 1000개 중 700개만)
반품거래처 반품 처리. 사유/수량 기록
출고 처리 순서
  1. 출고관리 > 출고 탭 이동
  2. 출고 가능 목록에서 해당 건 확인
  3. 출고 버튼 → 수량/차량번호 확인
  4. 거래명세서 인쇄 → 제품과 함께 전달
연동·동기화
  • 작업지시서(완료) → 출고: 작업지시서가 완료되어야 출고 가능 목록에 표시
  • 출고 → 매출: 출고 후 매출 등록 시 출고 내역 참조
  • 출고 → 세금계산서: 출고 기반으로 세금계산서 발행
  • 출고 → 재고: 출고 시 완제품 재고 자동 차감
  • 출고 → 거래명세서 인쇄: 거래명세서 자동 생성

2-4. 클레임

거래처 불만/불량 접수를 기록하고 처리 과정을 관리합니다.

클레임 등록
  1. 거래처, 품목, 유형(인쇄불량/치수불량/포장불량 등) 선택
  2. 내용: 어떤 문제인지 상세 기록
  3. 원인: 왜 발생했는지 (조사 후 기록)
  4. 대책: 재발 방지 조치 사항
  5. 상태: 미처리 → 처리중 → 완료
연동·동기화
  • 클레임 → 경영분석: 클레임 건수/추이가 KPI에 반영
  • 클레임 → 불량관리: 클레임 원인이 불량 분석 데이터로 활용

3. 생산관리

작업지시서를 중심으로 생산 계획, 공정 진행, 실적을 관리합니다. 작업지시서가 이 시스템의 핵심축입니다.

작업지시서 생성→ 자재 확인(BOM)→ 공정 진행→ 실적 입력→ 품질 검사→ 완료

3-1. 생산현황 (대시보드)

그룹탭내용
대시보드공장 전체 현황 한눈에 보기 — 오늘 작업 건수, 공정별 진행률(코팅/합지/톰슨/접착/인쇄), 완료/지연 건수, 일별 생산량 차트
작업자현황각 작업자의 현재 작업 내용, 오늘 실적(생산수량/불량수량)
연동·동기화
  • 작업지시서/공정실적 → 대시보드: 모든 데이터가 실시간 집계되어 KPI 카드와 차트에 표시
  • 공정실적 → 작업자현황: 작업자별 실적 입력 내역이 자동 집계

3-2. 생산계획

월/주 단위로 생산 일정을 계획합니다. 캘린더 형태로 작업을 배치하고, 납기일 기준으로 역산하여 각 공정의 시작일을 설정합니다.

3-3. 작업지시서 (핵심 기능)

모든 생산 활동의 중심입니다. 수주 → 작업지시서 → 공정 실적 → 완료 → 출고의 전체 흐름을 연결합니다.

작업지시서 등록
  1. 생산관리 > 작업지시서 또는 상단 + 작업지시서 버튼
  2. 거래처 선택 (자동완성)
  3. 품목 선택
  4. 목형 선택 (톰슨 공정 금형)
  5. 수량 입력
  6. 납기일 설정
  7. 공정 설정: 필요한 공정을 순서대로 체크 (예: 코팅 → 합지 → 톰슨 → 접착)
  8. 저장

작업지시서 상태 흐름

대기→ 진행중→ 완료대기→ 완료
상태의미전환 조건
대기생산 미시작등록 직후 기본값
진행중공정 진행 중작업 시작 또는 첫 공정 실적 입력 시
완료대기모든 공정 완료, 확인 대기마지막 공정 실적 입력 시
완료생산 완료, 출고 가능완료 처리 버튼 클릭
연동·동기화 (작업지시서 중심)
  • 견적/수주 → 작업지시서: 견적 수주 확정 시 자동 생성
  • 작업지시서 → BOM/MRP: 작업지시서의 품목/수량으로 필요 자재 자동 계산
  • 작업지시서 → 공정실적: 작업지시서 선택 → 각 공정별 실적 입력
  • 작업지시서 → 대시보드: 진행 상태가 실시간으로 대시보드에 반영
  • 작업지시서(완료) → 출고: 완료된 지시서만 출고 가능 목록에 표시
  • 작업지시서 → 인쇄: 현장 게시용 작업지시서 인쇄
  • 작업지시서 → 목형 사용이력: 사용한 목형의 사용 횟수 자동 증가
  • 작업지시서 → 납기관리: 납기일 기반 자동 알림

3-4. 공정실적

현장 작업자가 각 공정 완료 시 실적을 입력합니다.

그룹탭기능
실적입력작업지시서 선택 → 공정/작업자/생산수량/불량수량/작업시간 입력
로트추적로트번호로 "이 제품이 어떤 원자재로, 어떤 공정을, 누가 작업했는지" 역추적
연동·동기화
  • 공정실적 → 작업지시서 상태: 실적 입력에 따라 작업지시서 진행률 자동 갱신
  • 공정실적 → 불량관리: 불량수량 입력 시 불량관리에 자동 반영
  • 공정실적 → 대시보드: 공정별 생산량/불량률 실시간 집계
  • 공정실적 → OEE: 작업시간 데이터가 설비 OEE 계산에 사용
  • 공정실적 → 작업자현황: 작업자별 오늘 실적 자동 집계

3-5. 외주관리

인쇄 등 자체 설비가 없는 공정을 외부 인쇄소에 맡기는 과정을 관리합니다.

외주 발주 ~ 입고
  1. 생산관리 > 외주관리 > + 외주발주
  2. 인쇄소 선택 (마스터에 등록된 인쇄소)
  3. 작업지시서/품목/수량/납기 입력
  4. 인쇄소에서 완성품 도착 → 입고 처리
연동·동기화
  • 외주 → 작업지시서: 외주 공정 완료 시 작업지시서 진행률에 반영
  • 외주 → 매입: 외주비가 매입으로 연결
  • 외주 → 인쇄소 마스터: 등록된 인쇄소 목록에서 선택

4. 품질/설비

제품 품질 검사, 불량 관리, 공장 설비 유지보수를 관리합니다.

4-1. 검사관리

그룹탭언제 사용내용
수입검사원자재 입고 시원지/잉크 등 원자재가 들어올 때 품질 기준(평량/수분/색상 등)에 맞는지 검사. 합격/불합격 판정
출하검사제품 출고 전거래처에 보내기 전 최종 품질 체크. 치수/인쇄 상태/접착 강도/외관 검사
검사성적서거래처 요청 시공식 검사 성적서 발행. 거래처에 품질 증빙으로 제출하는 문서
연동·동기화
  • 입고 → 수입검사: 자재 입고 시 수입검사 대상으로 등록
  • 작업지시서(완료) → 출하검사: 생산 완료 건이 출하검사 대상으로 연결
  • 검사 → 출고: 출하검사 합격 후 출고 처리 가능

4-2. 불량관리

항목계산식예시
가동률실제 가동시간 / 계획 가동시간8시간 중 7시간 = 87.5%
성능률실제 생산량 / 이론 생산량이론 1000개, 실제 800개 = 80%
양품률양품 수량 / 총 생산량800개 중 760개 양품 = 95%
OEE가동률 x 성능률 x 양품률87.5% x 80% x 95% = 66.5%
세계적 수준의 OEE는 85% 이상입니다. 60% 미만이면 개선이 필요합니다.

6-2. 세금계산서

종이와 전자 두 가지 방식을 지원합니다.

그룹탭내용

전자세금계산서전자 발행 건만 모아보기. 국세청 전송 상태 확인

종이 세금계산서

종이 발행 순서
  1. + 등록 > 발행방식: 종이 (인쇄) 선택
  2. 발행일, 구분(매출/매입), 거래처, 사업자번호, 품목 입력
  3. 수량 x 단가 → 공급가액 자동 계산, 세액(10%) 자동
  4. 저장 후 종이 인쇄 버튼으로 인쇄

전자 세금계산서 (팝빌 API)

전자 발행 순서
  1. + 등록 > 발행방식: 전자 (국세청) 선택
  2. 모든 필드 입력 (대표자명, 주소 필수)
  3. 저장 → 상태: 전자대기
  4. 전자발행 버튼 → 팝빌 API로 국세청 전송
  5. 성공 시 → 전자발행 상태로 변경

홈택스 엑셀 내보내기 (무료 방법)

세금계산서 목록에서 홈택스 엑셀 버튼 → CSV 다운로드 → 홈택스(hometax.go.kr)에 업로드하면 무료로 전자세금계산서 발행 가능

팝빌: 건당 100원, 자동 전송. 홈택스: 무료, 수동 업로드. 발행 건수가 적으면 홈택스, 많으면 팝빌이 편합니다.

팝빌 API 설정

환경설정 > 설정 > 팝빌 전자세금계산서 API 설정에서 LinkID, SecretKey, 사업자번호를 입력합니다. (popbill.com에서 회원가입 후 발급)
