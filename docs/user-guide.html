0-1. 처음 사용할 때 — 기초 데이터 등록 순서

시스템을 처음 사용하기 전에 아래 순서로 기초 데이터를 등록해야 합니다.

1. 회사정보→ 2. 직원 등록→ 3. 거래처 등록→ 4. 품목 등록→ 5. 목형 등록→ 6. 설비 등록
Step 1. 회사정보 설정
  1. 시스템 > 시스템관리 > 설정 탭 이동
  2. 회사명, 대표자명, 사업자등록번호, 업태, 종목 입력
  3. 세금계산서 발행에 필요한 정보이므로 정확하게 입력
  4. 저장 클릭
Step 2. 직원 등록
  1. 회계/인사 > 인사관리 > + 직원등록
  2. 이름, 부서(현장/사무), 직급, 담당공정 선택
  3. 담당공정: 코팅, 합지, 톰슨, 접착, 인쇄 중 선택 (현장 작업자인 경우)
  4. 기본급, 시급 입력 → 급여 계산에 사용됨
Step 3. 거래처 등록
  1. 시스템 > 마스터관리 > 거래처 탭
  2. + 거래처 클릭
  3. 거래처명, 사업자등록번호(조회 버튼으로 확인 가능), 연락처, 주소 입력
  4. 거래 유형: 납품처(제품을 파는 곳) / 매입처(원자재 사는 곳) 선택
Step 4. 품목 등록
  1. 시스템 > 마스터관리 > 품목 탭
  2. + 품목 클릭
  3. 품목명, 규격, 재질, 단가, 거래처 입력
  4. 예: "A사 피자박스 / 400×300×50 / E골 / 350원"
Step 5. 목형 등록
  1. 시스템 > 마스터관리 > 목형 탭
  2. + 목형 클릭
  3. 목형명, 거래처, 사이즈, 상태(사용중/보관/폐기) 입력
  4. 목형 = 톰슨 공정에서 사용하는 금형 (박스 모양 찍는 틀)
Step 6. 설비 등록
  1. 품질/설비 > 설비관리 > 설비대장 탭
  2. + 설비등록 클릭
  3. 설비명(예: 코팅기 1호), 제조사, 구매일, 상태 입력
기초 데이터가 없으면 작업지시서, 견적, 출고 등에서 거래처/품목을 선택할 수 없습니다. 반드시 먼저 등록하세요.

1. 영업관리

고객 문의부터 견적 → 수주 → 출고까지의 영업 흐름을 관리합니다.

견적→ 수주 확정→ 작업지시서→ 생산→ 출고

견적관리

거래처에서 "이런 박스 만들어달라"는 요청이 오면 견적을 작성합니다.

견적 등록하기

견적 등록 순서
  1. 영업관리 > 견적관리 이동
  2. + 견적등록 클릭
  3. 자동으로 견적번호 부여됨 (QT-20260411-001 형식)
  4. 거래처: 요청한 거래처 입력
  5. 제품명: 만들 제품 이름 (예: "피자박스 대")
  6. 수량: 주문 수량 (매 단위)
  7. 확정 공급가: 최종 판매 단가 (마진 포함)
  8. 클로드 견적 내용: 원가 계산한 내용 붙여넣기 (선택)
  9. 저장 클릭

"클로드 견적 내용" 칸이란?

클로드(AI)에게 "E골 400×300×50 박스 1000매 원가 계산해줘"라고 물어본 결과를 통째로 복사해서 붙여넣는 곳입니다. 원지비, 코팅비, 합지비, 톰슨비, 접착비 등의 원가 내역이 기록으로 남습니다.

견적 상태 변경

상태의미언제 바꾸나
작성중아직 거래처에 보내지 않음처음 등록 시 기본값
발송거래처에 견적서를 보냄이메일/팩스로 견적서 전달 후
수주거래처가 OK, 주문 확정거래처에서 발주 확인 후
실주거래처가 거절, 수주 실패다른 업체에 빼앗겼을 때

견적서 인쇄

견적 목록에서 인쇄 버튼 클릭 → 공식 견적서 양식으로 새 창이 열리고 인쇄됩니다.

견적 → 작업지시서 전환

상태를 수주로 바꾸면 → 작업지시서 전환 버튼이 나타납니다. 클릭하면 견적 정보가 자동으로 작업지시서에 복사됩니다.

수주관리

확정된 수주(주문)을 관리합니다.

그룹탭

탭기능
수주목록전체 수주 목록. 상태별 필터링 가능
진행추적수주별로 현재 어느 단계에 있는지 추적 (생산 진행률, 출고 여부)

출고관리

생산 완료된 제품을 거래처에 납품하는 과정을 관리합니다.

그룹탭

탭기능

부분출고전체 수량 중 일부만 먼저 출고할 때 사용
반품거래처에서 제품을 돌려보냈을 때 처리
출고 처리 방법
  1. 출고관리 > 출고 탭 이동
  2. 출고 가능 목록에서 해당 건 확인
  3. 출고 버튼 클릭
  4. 출고 수량, 차량번호, 비고 확인
  5. 거래명세서 인쇄 → 인쇄해서 제품과 함께 전달

출고 후 자동 처리

  • 출고 이력에 자동 기록
  • 매출/매입 메뉴에서 매출로 연결 가능
  • 세금계산서 발행 시 출고 내역 참조

클레임

거래처에서 불량/문제가 있다고 연락 올 때 기록합니다.

클레임 등록
  1. 영업관리 > 클레임 이동
  2. + 클레임등록 클릭
  3. 거래처, 품목, 유형(인쇄불량/치수불량/포장불량 등) 선택
  4. 내용: 어떤 문제인지 상세 기록
  5. 원인: 왜 발생했는지 (조사 후 기록)
  6. 대책: 어떻게 조치했는지
  7. 상태: 미처리 → 처리중 → 완료
클레임 기록은 품질 개선의 근거자료가 됩니다. 경영분석에서 클레임 추이를 확인할 수 있습니다.

2. 생산관리

작업지시서를 중심으로 생산 계획, 공정 진행, 실적을 관리합니다.

작업지시서 생성→ 공정 진행→ 실적 입력→ 완료→ 출고 대기

생산현황

대시보드

공장 전체 현황을 한눈에 보는 화면입니다.

  • 오늘의 작업: 진행중/완료/지연 건수
  • 공정별 현황: 코팅, 합지, 톰슨, 접착, 인쇄 각 공정의 진행 상태
  • 차트: 일별/주별 생산량 추이

작업자현황

각 작업자가 현재 어떤 작업을 하고 있는지, 오늘 실적이 얼마인지 확인합니다.

생산계획

월/주 단위로 생산 일정을 계획합니다. 캘린더 형태로 작업을 배치하고 납기일을 관리합니다.

납기일이 빨간색으로 표시되면 지연 위험입니다. 생산 일정을 조정하세요.

작업지시서

이 시스템의 가장 핵심 기능입니다. 모든 생산 활동은 작업지시서를 기준으로 진행됩니다.

작업지시서 등록
  1. 생산관리 > 작업지시서 또는 상단 + 작업지시서 버튼
  2. 거래처 선택 (입력하면 자동완성 목록 표시)
  3. 품목 선택
  4. 목형 선택 (톰슨 공정에 사용할 금형)
  5. 수량 입력
  6. 납기일 설정
  7. 공정 설정: 이 제품에 필요한 공정을 순서대로 체크
    예: 코팅 → 합지 → 톰슨 → 접착
  8. 저장

작업지시서 상태 흐름

대기→ 진행중→ 완료대기→ 완료
상태의미전환 조건
대기아직 생산 시작 안 함등록 직후 기본값
진행중공정 진행 중작업 시작 버튼 클릭 또는 공정실적 입력 시
완료대기모든 공정 끝, 검사/확인 대기마지막 공정 실적 입력 완료 시
완료생산 완료, 출고 가능완료 처리 버튼 클릭
작업지시서가 완료 상태가 되어야 출고관리에서 출고 처리를 할 수 있습니다.

작업지시서 인쇄

작업지시서 상세 화면에서 인쇄 버튼 클릭 → 현장에 붙일 수 있는 공식 양식으로 인쇄됩니다.

공정실적

현장 작업자가 각 공정을 끝낼 때마다 실적을 입력합니다.

그룹탭

탭기능
실적입력작업지시서별 공정 실적 기록
로트추적로트번호로 제품의 생산 이력 역추적
공정 실적 입력
  1. 생산관리 > 공정실적 이동
  2. 작업지시서 선택 (드롭다운)
  3. 공정 선택 (예: 톰슨)

로트추적

로트번호란? 같은 시기에 같은 조건으로 생산된 제품 묶음의 식별 번호입니다.

거래처에서 "지난번 납품분 문제있어요"라고 할 때, 로트번호로 검색하면 어떤 원자재를 사용해서, 어떤 공정을, 누가 작업했는지 전부 추적할 수 있습니다.

외주관리

인쇄처럼 자체 설비가 없어서 외부 인쇄소에 맡기는 공정을 관리합니다.

외주 발주 ~ 입고
  1. 생산관리 > 외주관리 이동
  2. + 외주발주 클릭
  3. 인쇄소(마스터에서 등록한 인쇄소) 선택
  4. 작업지시서/품목/수량/납기 입력
  5. 인쇄소에서 완성품 도착하면 입고 처리

3. 품질/설비

제품 품질 검사, 불량 관리, 공장 설비 유지보수를 관리합니다.

검사관리

그룹탭

탭언제 사용설명
수입검사원자재 입고 시원지, 잉크 등 원자재가 들어올 때 품질 기준에 맞는지 검사. 합격/불합격 판정
출하검사제품 출고 전거래처에 보내기 전 최종 품질 체크. 치수, 인쇄 상태, 접착 강도 등

불량관리

공정 중 발생한 불량을 기록하고, 어떤 공정에서 불량이 많은지 분석합니다.

  • 원인 분석: 설비 문제? 자재 문제? 작업자 실수?

설비관리

공장 기계(코팅기, 합지기, 톰슨기, 접착기 등)를 관리합니다.

그룹탭 (6개)

탭설명
설비대장보유 설비 목록. 설비명, 제조사, 구매일, 현재 상태(가동/정지/점검) 관리
예방보전정기 점검 스케줄. 예: "톰슨기 매월 칼날 교체", "코팅기 분기별 롤러 점검"
고장이력기계가 고장났을 때 기록. 언제/어떤 설비가/왜 고장났는지/수리 내용
비가동기계가 안 돌아간 시간. 사유: 고장, 자재대기, 세척, 교대 등
OEE설비종합효율 = 가동률 × 성능률 × 양품률. 기계가 얼마나 효율적으로 돌아가는지 수치화
금형이력목형(톰슨금형)의 사용 횟수, 수리 이력. 수명 관리용

OEE란?

OEE(Overall Equipment Effectiveness) = 설비종합효율

항목계산예시
가동률실제 가동시간 / 계획 가동시간8시간 중 7시간 가동 = 87.5%
성능률실제 생산량 / 이론 생산량이론상 1000개인데 800개 생산 = 80%
양품률양품 수량 / 총 생산량800개 중 760개 양품 = 95%
OEE가동률 × 성능률 × 양품률87.5% × 80% × 95% = 66.5%
세계적 수준의 OEE는 85% 이상입니다. 60% 미만이면 개선이 필요합니다.

BOM/MRP

BOM (Bill of Materials, 자재명세서)

제품 하나를 만드는 데 필요한 자재 목록입니다.

BOM 등록 예시: "피자박스 1000매"
  1. BOM/MRP > BOM 탭 > + BOM등록
  2. 완성품: 피자박스
  3. 자재 추가:
    • E골 원지 — 500장
    • 인쇄 잉크 — 2L
    • 접착제 — 1kg
    • 코팅 필름 — 50m
  4. 저장

MRP (Material Requirements Planning, 자재소요계획)

현재 작업지시서를 기반으로 "부족한 자재가 뭔지" 자동 계산합니다.

  • 작업지시서의 생산 수량 × BOM의 자재 수량 = 필요 자재량
  • 필요 자재량 - 현재 재고 = 부족량 (= 발주 필요량)
  • 부족한 자재는 빨간색으로 표시되고, 바로 발주서 생성 가능
BOM을 정확히 등록해두면 MRP가 자동으로 발주량을 계산해줍니다. 자재 과다/부족 구매를 방지할 수 있습니다.

세금계산서

매출/매입에 대한 세금계산서를 발행합니다. 종이와 전자 두 가지 방식을 지원합니다.

그룹탭

탭설명
세금계산서전체 세금계산서 목록 (종이+전자 모두)
전자세금계산서전자 발행 건만 모아보기. 국세청 전송 상태 확인
세금계산서 등록 (종이)
  1. 회계/인사 > 세금계산서 이동
  2. + 등록 클릭
  3. 발행방식: 종이 (인쇄) 선택 (기본값)
  4. 발행일, 구분(매출/매입), 거래처, 사업자번호 입력
  5. 수량 × 단가 → 공급가액 자동 계산, 세액(10%) 자동
  6. 저장 후 종이 인쇄 버튼으로 인쇄
세금계산서 등록 (전자 — 팝빌 API)
  1. + 등록 > 발행방식: 전자 (국세청) 선택
  2. 모든 필드 입력 (대표자명, 주소 필수)
  3. 저장 → 상태가 전자대기로 저장
  4. 전자발행 버튼 클릭 → 팝빌 API를 통해 국세청에 전송
  5. 성공 시 전자발행 상태로 변경

팝빌 API 설정 (전자발행 사용 시)

팝빌 설정 방법
  1. popbill.com에서 사업자 회원가입
  2. 연동 설정에서 LinkID, SecretKey 발급
  3. 시스템 > 시스템관리 > 설정 > 팝빌 전자세금계산서 API 설정
  4. LinkID, SecretKey, 사업자등록번호 입력
  5. 테스트 모드 체크 (처음에는 테스트로 확인)
  6. 설정 저장

홈택스 엑셀 내보내기 (무료 방법)

팝빌 비용을 아끼려면 홈택스 엑셀 버튼으로 CSV 파일을 다운로드하고, 홈택스에 직접 업로드하면 무료로 전자세금계산서를 발행할 수 있습니다.

홈택스 업로드 방법
  1. 세금계산서 목록에서 홈택스 엑셀 버튼 클릭 → CSV 다운로드
  2. 홈택스 로그인
  3. 조회/발급 > 전자세금계산서 > 발급 > 일괄발급 업로드
  4. 다운받은 CSV 파일 업로드
  5. 확인 후 일괄 발급
팝빌: 건당 100원 과금, 자동 전송. 홈택스: 무료, 수동 업로드. 발행 건수가 적으면 홈택스, 많으면 팝빌이 편합니다.
